# 多智能体股票分析系统

> 版本：v3.0
> 定位：半自动化交易辅助系统（AI分析 + 人工确认）
> 架构：三层架构（数据层 → 工具层 → 智能体层）

---

## 二、智能体流程

### Stage 1：六维分析（并行）

六个分析师同时运行，各自从一个维度分析个股，互不干扰。

| 智能体 | 分析维度 | 核心指标 |
|-------|---------|---------|
| 技术分析师 | 价格与成交量行为 | MACD / RSI / 均线 / 布林带 |
| 基本面分析师 | 财务与估值 | PE / PB / 现金流 / 净利润 |
| 市场微观结构分析师 | 资金流向与筹码 | 主力净流入 / 龙虎榜 / 融资融券 |
| 市场情绪分析师 | 市场整体情绪 | 大盘涨跌停家数 / 换手率 / 北向资金 |
| 板块轮动分析师 | 所在板块强弱 | 板块动量 / 行业指数 / 龙头效应 |
| 资讯事件分析师 | 新闻与公告影响 | 公告内容 / 政策事件 / 突发消息 |

各分析师输出 0-100 评分的分析报告（Markdown格式）。

### Stage 2：多空研判与交易计划（串行）

| 角色 | 职责 |
|-----|------|
| 看涨分析师 | 基于六维分析提出看涨论点 |
| 看跌分析师 | 基于六维分析提出看跌论点，双方辩论1-2轮 |
| 研究主管 | 综合六维评分和多空辩论，给出方向和置信度 |
| 交易计划师 | 将研究主管裁决转化为具体交易参数（入场/止损/止盈/仓位/持有期） |

### Stage 3：量化风控与最终裁决（混合）

```
执行顺序：
  Step 1（纯代码）：风控计算模块运行 → VaR计算 + A股特有风险评分
     ↓ 计算结果注入以下4位AI智能体
  Step 2（AI并行辩论）：三位风控师各自评估，形成三方意见
     ↓
  Step 3（AI裁决）：首席风控官综合三方，给出最终裁决

注：Stage 3 提示词文件编号从 _03 开始（_03~_06），
    _01/_02 位置对应 Step 1 的纯代码计算模块，不是AI提示词，
    详见 docs/tools/03_风控计算模块.md
```

| 角色 | 文件编号 | 职责 |
|-----|---------|------|
| [纯代码] VaR计算 | _01（工具层） | 历史模拟法计算持仓风险敞口 |
| [纯代码] A股风险评分 | _02（工具层） | 涨跌停/T+1/融资/停牌特有风险打分 |
| 激进风控师 | _03 | 从机会成本视角评估，关注收益潜力 |
| 保守风控师 | _04 | 从本金保护视角评估，关注下行风险 |
| 量化风控师 | _05 | 基于 VaR 和数学模型给出客观评估 |
| 首席风控官 | _06 | 综合三方意见，拥有一票否决权，给出最终裁决 |

### Stage 4：投资顾问（用户可读报告）

```
输入：CRO最终裁决 + 交易计划书 + Stage 1 六维评分汇总 + 价格快照

输出：面向普通投资者的交易建议报告
  ├── 单一操作建议（买入/卖出/持仓/观望，唯一明确）
  ├── 明日价格参考区间（基于技术支撑/阻力 + 波动率）
  ├── 7日价格走势参考区间（基于趋势方向 + 关键位）
  ├── 关键数字（止损价/目标价/建议仓位，全为精确数字）
  ├── 3条核心理由（无专业术语，小白可读）
  ├── 2条主要风险提示（醒目位置）
  └── 分步操作指南（具体步骤，小白可执行）
```
