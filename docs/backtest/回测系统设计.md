# 回测系统设计

> 版本：v1.0
> 模块状态：可选模块，默认不集成，按需启用
> 所属层：独立于三层主架构，通过接口与主系统交互

---

## 一、定位与目的

### 1.1 回测回答的核心问题

"这套多智能体分析流程，在过去5年的历史数据上，能否产生正向超额收益？"

即：假设从某一历史时间点开始，每天按本系统的分析规则做出交易决策，最终的资金曲线表现如何。

### 1.2 与主系统的关系

```
主系统（在线，实时）：
  实时数据 → Stage 1 → Stage 2 → Stage 3 → 交易建议

回测系统（离线，历史）：
  历史数据 → 信号规则化 → 回测引擎 → 绩效报告
```

**关键设计原则**：
- 回测与主系统完全解耦，回测不运行时不影响主系统任何功能
- 回测时不能使用 AI 实时推理（速度和成本均不可行），必须将分析逻辑**规则化**为确定性计算
- 回测结果用于评估策略有效性，不用于修改主系统的实时判断

---

## 二、为什么默认不集成

| 原因 | 说明 |
|-----|------|
| 优先级 | 先保证主系统能正常运行，再考虑历史验证 |
| 资源消耗 | 回测需要大量历史数据计算，耗时较长 |
| 复杂性 | 需要处理未来函数、幸存者偏差、滑点等细节问题 |
| 使用场景 | 主系统作为辅助决策工具已可用，回测是进阶需求 |

**建议启用时机**：
- 系统运行一段时间后，想量化评估实际效果
- 调整策略参数前，想验证新参数是否更优
- 准备增加投入前，想做全面的历史压力测试

## 三、信号规则化设计

回测时将 AI 分析逻辑转换为确定性的量化规则，保证结果可重复、可追溯。

### 3.1 Stage 1 规则化（技术评分）

以50分为基准，按以下规则加减分后限制在0-100范围内：

| 条件 | 加减分 |
|-----|-------|
| MACD金叉（当日或近3日内） | +15 |
| MACD死叉 | -15 |
| DIF > 0 且 DEA > 0（零轴上方） | +10 |
| DIF < 0 且 DEA < 0 | -10 |
| RSI 在 50-70（强势中性区） | +10 |
| RSI > 80（严重超买） | -10 |
| RSI < 30（超卖反弹区） | +5 |
| 均线多头排列（MA5>MA10>MA20>MA60） | +15 |
| 均线空头排列 | -15 |
| 当日上涨且量比 > 2（放量上涨） | +10 |
| 当日下跌且量比 > 2（放量下跌） | -10 |

其他维度（基本面、资金面、情绪面、板块面）同样依据计算引擎中的评分规则直接计算，不通过 AI 推理。

### 3.2 Stage 2 规则化（多空判断）

六维评分按标准权重加权合并：

| 维度 | 权重 |
|-----|-----|
| 技术面 | 20% |
| 基本面 | 20% |
| 微观结构（资金面） | 25% |
| 情绪面 | 15% |
| 板块效应 | 15% |
| 资讯事件 | 5%（历史回测中固定使用中性值50） |

**方向判断规则**：

| 综合加权评分 | 方向 | 置信度 |
|-----------|------|-------|
| ≥ 65 | 看涨（bullish） | 等于综合评分 |
| ≤ 40 | 看跌（bearish） | 100 - 综合评分 |
| 40-65 | 中性（neutral） | 固定50 |

### 3.3 Stage 3 规则化（风控过滤）

| 检查项 | 触发条件 | 处理方式 |
|-------|---------|---------|
| VaR 过高 | 持有期VaR > 持仓市值10% | 按比例降仓至VaR降至8%以内 |
| A股特有风险高 | 综合风险评分 ≥ 70 | 否决本次交易 |
| A股特有风险偏高 | 综合风险评分 50-69 | 仓位降至原计划70% |

## 四、回测引擎流程

### 4.1 整体步骤

| 步骤 | 内容 |
|-----|------|
| 初始化 | 设定回测区间（建议5年）、股票池（如沪深300成分股）、初始资金 |
| 逐日遍历 | 对每个交易日：数据准备 → 信号生成 → 风控过滤 → 订单执行 → 状态记录 |
| 绩效计算 | 全周期结束后计算各绩效指标 |
| 生成报告 | 输出标准格式回测报告 |

### 4.2 未来函数防范规则

所有信号生成只能使用目标交易日**前一日及之前**的历史数据，绝对禁止使用当日收盘后才产生的数据来生成当日信号。这是回测有效性的底线。

### 4.3 订单执行规则

| 规则项 | 说明 |
|-------|------|
| 买入时机 | 信号日次日开盘价买入（T+1制度，不可当日建仓后当日读取收盘信号） |
| 卖出时机 | 触发止损或止盈时次日开盘卖出 |
| 单股仓位 | 默认计划仓位10%，经风控过滤后可降至7% |
| 止损线 | 入场价下方8% |
| 止盈线 | 入场价上方15% |
| 最大同时持仓 | 不超过10只（避免仓位过于分散） |

## 六、绩效指标体系

### 6.1 核心指标

| 指标 | 计算方式 | 合格标准 | 优秀标准 |
|-----|---------|---------|---------|
| 年化收益率 | (1 + 总收益) ^ (252/回测天数) - 1 | > 10% | > 20% |
| 最大回撤 | 回测期内最大净值下降幅度 | < 20% | < 15% |
| 夏普比率 | 年化超额收益 / 年化波动率（无风险利率3%） | > 1.0 | > 1.5 |
| 卡玛比率 | 年化收益率 / 最大回撤绝对值 | > 0.5 | > 1.0 |
| 胜率 | 盈利交易数 / 总已平仓交易数 | > 45% | > 55% |
| 盈亏比 | 平均盈利金额 / 平均亏损金额 | > 1.2 | > 1.5 |

### 6.2 回测报告标准章节

| 章节 | 内容 |
|-----|------|
| 回测设置 | 区间、股票池、初始资金、交易成本参数 |
| 核心指标对比 | 策略 vs 基准（沪深300）的全部指标及超额收益 |
| 收益曲线 | 策略净值曲线 vs 基准净值曲线 |
| 回撤分析 | 最大回撤数值、发生区间、修复所需天数 |
| 交易统计 | 总交易次数、盈亏次数、最大单笔盈亏 |
| 月度分布 | 按月展示收益热力图或表格 |
| 结论与建议 | 策略有效性评价、需优化点、参数敏感性分析 |

## 七、常见陷阱

| 陷阱 | 说明 | 规避方法 |
|-----|------|---------|
| 未来函数 | 用T日收盘后数据生成T日信号 | 严格使用 T-1 及之前数据 |
| 幸存者偏差 | 只回测现存股票，遗漏已退市标的 | 使用含退市股的完整历史股票池 |
| 过拟合 | 参数过度优化历史数据，样本外失效 | 样本外测试 + 滚动回测 |
| 滑点低估 | 假设按收盘价成交 | 添加滑点模型（买入加价0.1%，卖出减价0.1%） |
| 流动性忽视 | 假设大单能立即成交 | 添加成交量约束，超过日均量5%的仓位分批建仓 |
