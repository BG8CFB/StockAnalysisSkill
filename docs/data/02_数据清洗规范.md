# 数据清洗规范

> 版本：v1.0
> 所属层：数据层（三层架构第一层）
> 职责范围：定义数据从原始状态到标准化状态的全部处理规则，不涉及指标计算

---

## 一、整体流程

```
┌──────────────────────────────────────────────────────────────┐
│                     数据清洗流程                               │
│                                                              │
│  原始数据（来自适配层标准字段）                                  │
│            ↓                                                 │
│  ┌──────────────────────┐                                    │
│  │  Step 1: 前复权处理   │  开/高/低/收价格统一为前复权价格       │
│  │                      │  成交量不复权，涨跌幅使用原始值         │
│  └─────────┬────────────┘                                    │
│            ↓                                                 │
│  ┌──────────────────────┐                                    │
│  │  Step 2: 停牌检测     │  ← 【硬终止节点】                   │
│  │                      │                                    │
│  │  当日停牌？           │                                    │
│  │   是 ──────────────────────────────────────────────→      │
│  │   （立即终止清洗，                                          │
│  │    输出：is_suspended=true，终止分析流程，                  │
│  │    Stage 1 所有智能体不启动）                               │
│  │   否 ↓                                                    │
│  └─────────┬────────────┘                                    │
│            ↓                                                 │
│  ┌──────────────────────┐                                    │
│  │  Step 3: 异常值检测   │  标注涨跌停/量价异常/跳空            │
│  │                      │  数据保留，不删除，加事件标签          │
│  └─────────┬────────────┘                                    │
│            ↓                                                 │
│  ┌──────────────────────┐                                    │
│  │  Step 4: 缺失值处理   │  字段缺失→输出null，标记原因         │
│  │                      │  不估算，不补全                      │
│  └─────────┬────────────┘                                    │
│            ↓                                                 │
│  标准化数据包 → 计算引擎                                       │
│  （含 metadata / price_series / anomalies / missing_fields）  │
└──────────────────────────────────────────────────────────────┘
```

## 二、前复权处理

### 2.1 为什么必须复权

分红、配股、送转会导致价格产生向下跳空，导致以下问题：
- 均线、MACD 等趋势指标在除权日前后出现虚假断层
- 跨越除权日的涨跌幅计算失真
- 历史支撑阻力位因价格跳变而失效

因此，所有基于价格序列的计算均须使用**前复权价格**。

### 2.2 复权计算规则

**计算方式**：以最新一日复权因子为基准，对历史全部价格进行等比缩放。

**处理字段**：开盘价、最高价、最低价、收盘价，分别输出对应的 `open_adj`、`high_adj`、`low_adj`、`close_adj`。

**注意事项**：
- 成交量不参与复权，使用原始值
- 涨跌幅（`pct_chg`）使用原始值，反映实际当日涨跌
- 每日重新计算复权价格，以当日最新复权因子为基准

### 2.3 复权因子缺失处理

| 情形 | 处理方式 | 质量标记 |
|-----|---------|---------|
| 复权因子正常 | 按公式计算前复权价格 | — |
| 复权因子接口超时 | 使用未复权原始价格，降级使用 | 注明"价格未复权，趋势指标参考价值下降" |
| 复权因子数据不完整（历史缺失） | 对有因子的区间复权，缺失区间标注 | 注明缺失日期范围 |

## 三、停牌检测

### 3.1 检测逻辑

调用数据源停牌信息接口，判断当日是否处于停牌状态。停牌时，行情数据（OHLCV）为无效数据，不得纳入任何指标计算。

### 3.2 停牌类型与风险等级

| 停牌类型 | 典型原因 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 重大事项停牌 | 重大重组、并购、定增 | 高 | 停牌时间通常较长（数周至数月），持仓风险极高 |
| 股价异常波动停牌 | 价格异常涨跌触发监管关注 | 中 | 通常1-3天，复牌后可能补涨/补跌 |
| 临时停牌 | 重大公告发布、技术原因 | 低 | 当日内恢复，影响有限 |

### 3.3 停牌时的处理规则

- 当日停牌：标注 `is_suspended = true`，跳过所有指标计算，不产生分析报告
- 历史数据中含停牌日：停牌日数据点剔除，不纳入均线等历史窗口计算
- 长期停牌后复牌：复牌首日数据单独标注，不与停牌前数据直接衔接计算趋势

## 四、异常值检测

### 4.1 检测项目清单

对每个交易日逐项扫描，产出异常事件列表，每条记录包含：日期、异常类型、描述。

| 检测项 | 判断条件 | 异常类型标签 |
|-------|---------|------------|
| 涨停 | 涨幅 ≥ 9.5%（主板） / ≥ 19%（科创/创业板），且收盘价等于最高价 | `limit_up` |
| 跌停 | 涨幅 ≤ -9.5%（主板） / ≤ -19%（科创/创业板），且收盘价等于最低价 | `limit_down` |
| 一字涨停板 | 开盘=最高=最低=收盘，处于涨停状态 | `one_word_limit_up` |
| 一字跌停板 | 开盘=最高=最低=收盘，处于跌停状态 | `one_word_limit_down` |
| 成交量异常 | 当日成交量 > 20日平均成交量 × 5 | `volume_surge` |
| 价格跳空 | 当日开盘价与前日收盘价之差超过 3% | `gap_open` |

### 4.2 异常数据的处理原则

- 涨跌停日：数据保留，异常标注，但不剔除。指标计算正常进行，计算结果需附带异常标注
- 一字板日：价格无波动，布林带等需要价格方差的指标在该日输出为 null，标注原因
- 量价异常日：数据保留，异常事件写入 `anomalies` 列表，供智能体分析时参考

## 五、缺失值处理规则

| 缺失情形 | 处理方式 | 字段标记 |
|---------|---------|---------|
| 历史数据不足（如上市不足一个计算周期） | 对应指标输出 `null`，不参与评分 | `insufficient_history` |
| 接口超时或数据返回错误 | 重试3次，仍失败则字段输出 `null` | `api_timeout` |
| 字段存在但数值异常（如负成交量） | 字段输出 `null`，异常写入日志 | `data_anomaly` |
| 复权因子缺失导致无法复权 | 使用原始价格，标注未复权 | `not_adjusted` |

## 六、清洗后数据输出格式

清洗完成后，输出标准化数据包交给计算引擎，包含以下部分：

| 数据段 | 内容 |
|-------|------|
| `metadata` | 股票代码、名称、分析日期、数据获取时间、数据来源、数据质量等级 |
| `market_status` | 当日行情：是否停牌、开高低收、涨跌幅、成交量、成交额（均为原始值） |
| `price_series` | 历史前复权价格序列（日期、开高低收复权价、成交量），供计算引擎使用 |
| `daily_basic` | 每日基本面指标（PE/PB/换手率/流通市值等） |
| `capital_flow_raw` | 原始资金流向数据（多日，仅A股） |
| `margin_raw` | 原始融资融券数据 |
| `anomalies` | 异常事件列表（日期、类型、描述） |
| `missing_fields` | 缺失或质量异常的字段列表 |

## 七、性能要求

| 指标 | 要求 |
|-----|------|
| 单只股票清洗时间 | 小于 2 秒 |
| 批量处理支持 | 最多 50 只股票并行 |
| 内存占用 | 单只股票小于 100MB |
