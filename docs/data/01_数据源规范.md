# 数据源规范

> 版本：v1.0
> 所属层：数据层（三层架构第一层）
> 职责范围：定义数据源接入原则、各数据源覆盖范围、字段映射规范

---

## 一、设计原则与整体架构

**数据源适配工具，而非工具适配数据源。**

工具层定义统一的标准字段体系，所有数据源以适配器模式对接工具层。新增或切换数据源只需编写该数据源的适配逻辑，工具层和智能体层不受任何影响。

### 1.1 整体数据流

```
┌─────────────────────────────────────────────────────────────────┐

│                                                                 │
```

## 二、当前支持的数据源

### 2.1 Tushare Pro（主数据源）

- 定位：付费数据源，数据质量高，覆盖全面
- 积分要求：5000积分及以上
- 请求频率：500次/分钟（5000积分级别）
- 适用市场：A股为主，部分港股数据

**数据覆盖范围：**

| 数据类别 | 是否可用 | 说明 |
|---------|---------|------|
| 日线行情（OHLCV） | 可用 | 主板、科创板、创业板、北交所 |
| 复权因子 | 可用 | 前复权、后复权计算基础 |
| 每日指标（PE/PB/换手率） | 可用 | 每日收盘后更新 |
| 主力资金流向 | 可用 | T+0，收盘后约1小时 |
| 北向资金（陆股通） | 可用 | 每日 |
| 融资融券数据 | 可用 | 每日 |
| 龙虎榜数据 | 可用 | 每日 |
| 涨跌停统计 | 可用 | 每日 |
| 停牌信息 | 可用 | 实时 |
| 财务三表 | 可用 | 季度更新 |
| 概念/行业板块 | 可用 | 不定期更新 |
| 行业指数行情 | 可用 | 每日 |
| 利率数据（SHIBOR） | 可用 | 每日 |
| 新闻资讯 | 需额外付费 | 约1000元/年 |
| 分钟数据 | 需额外付费 | 约2000元/年 |

**缓存策略：**
- 日线数据：缓存1小时
- 财务数据：缓存1天（季度数据变化频率低）
- 停牌信息：实时查询，不缓存

### 2.2 AkShare（免费备用数据源）

- 定位：开源免费数据源，无需 Token
- 稳定性：较 Tushare 低，接口可能随时变更
- 适用场景：Tushare 付费接口缺失时的补充，或低成本调试环境
- 适用市场：A股、港股、美股均有一定覆盖

**数据覆盖范围（与 Tushare 互补部分）：**

| 数据类别 | 说明 |
|---------|------|
| 日线行情 | 可用，稳定性中等 |
| 基本面数据 | 覆盖部分财务指标 |
| 新闻资讯 | 东方财富、同花顺等公开新闻（免费） |
| 宏观数据 | 覆盖较广，但更新不及时 |

### 2.3 扩展数据源

系统预留扩展接口，后续可接入其他数据源（如万得、聚宽等），只需实现标准适配层即可，无需修改工具层或智能体层。

## 三、标准字段体系

数据源字段经适配层统一映射为以下标准字段。智能体和工具层只使用标准字段名，不接触数据源原始字段名。

### 3.1 行情字段（标准）

| 标准字段名 | 含义 | 单位 |
|----------|------|------|
| `ts_code` | 股票代码（含市场后缀，如 000001.SZ） | — |
| `trade_date` | 交易日期 | YYYYMMDD |
| `open_adj` | 前复权开盘价 | 元 |
| `high_adj` | 前复权最高价 | 元 |
| `low_adj` | 前复权最低价 | 元 |
| `close_adj` | 前复权收盘价 | 元 |
| `vol` | 成交量 | 手（100股） |
| `amount` | 成交额 | 万元 |
| `pct_chg` | 涨跌幅 | % |

### 3.2 基本面字段（标准）

| 标准字段名 | 含义 | 说明 |
|----------|------|------|
| `pe_ttm` | 市盈率（TTM） | 为负或数据缺失时为 null |
| `pb_mrq` | 市净率（MRQ） | — |
| `ps_ttm` | 市销率（TTM） | — |
| `dividend_yield` | 股息率 | % |
| `turnover_rate` | 换手率 | % |
| `circ_mv` | 流通市值 | 万元 |

### 3.3 资金流向字段（标准）

| 标准字段名 | 含义 | 说明 |
|----------|------|------|
| `main_net_today` | 当日主力净流入 | 万元，A股专属 |
| `main_net_3d` | 近3日主力净流入累计 | 万元 |
| `main_net_5d` | 近5日主力净流入累计 | 万元 |
| `main_net_10d` | 近10日主力净流入累计 | 万元 |
| `retail_net_today` | 当日散户净流入 | 万元 |
| `main_ratio` | 主力资金占成交额比例 | % |
| `northbound_flow` | 北向资金净流入 | 万元，A股专属，其他市场为 null |

### 3.4 融资融券字段（标准）

| 标准字段名 | 含义 | 说明 |
|----------|------|------|
| `margin_balance` | 融资余额 | 万元 |
| `margin_ratio` | 融资余额占流通市值比例 | % |
| `margin_change_5d` | 5日融资余额变动幅度 | % |

### 3.5 停牌与风险字段（标准）

| 标准字段名 | 含义 | 说明 |
|----------|------|------|
| `is_suspended` | 当日是否停牌 | bool |
| `suspend_type` | 停牌类型 | 重大事项 / 股价异动 / 其他 |
| `risk_level` | 风险等级 | high / medium / low |

## 四、数据质量等级

工具层在注入数据时，会附加数据质量说明，便于智能体评估可信度。

| 质量等级 | 含义 | 智能体应对方式 |
|---------|------|-------------|
| `complete` | 所有关键字段有效 | 正常分析 |
| `partial` | 部分字段缺失或质量低 | 降低缺失维度权重，说明缺失项 |
| `insufficient_history` | 历史数据不足（如新上市） | 注明"数据不足，结论参考价值有限" |
| `api_timeout` | 接口超时，使用缓存数据 | 注明数据时效，降低置信度 |
| `suspended` | 股票停牌，无有效行情数据 | 直接标注停牌，终止分析 |

## 五、注入格式说明

工具层在将数据注入智能体上下文时，会在数据头部附加以下说明信息：

- 股票代码和名称
- 数据获取时间
- 数据来源（Tushare / AkShare / 其他）
- 数据质量等级
- 缺失字段列表（如有）

智能体在报告中引用数据时，须基于此注入的实际数据，不得使用训练知识中的估算值。
