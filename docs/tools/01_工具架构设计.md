## 一、工具层在系统中的位置

```
┌─────────────────────────────────────────────────────────────┐
│                       三层架构总览                           │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  数据层                                               │   │
│  │  数据源 → 清洗引擎 → 计算引擎 → 标准化数据输出         │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  工具层  ← 【本文档范围】                              │   │
│  │                                                     │   │
│  │   类型A：数据工具（自动注入）                          │   │
│  │   类型B：Skills技能（智能体主动调用）                  │   │
│  │   类型C：风控计算模块（纯代码，见 03文档）             │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  智能体层                                             │   │
│  │  Stage 1（并行）→ Stage 2（串行）→ Stage 3 → Stage 4  │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘

工具层职责：数据传递 + 能力注册
工具层不做：业务逻辑判断、投资决策、数据计算（计算由数据层完成）
```

## 二、两类工具的核心区别

```
┌──────────────────────────────────────────────────────────────┐
│                    工具二元分类总览                            │
│                                                              │
│  ┌───────────────────────┐   ┌───────────────────────────┐  │
│  │   类型A：数据工具       │   │   类型B：Skills 技能        │  │
│  │   (Data Tools)        │   │   (Skill Tools)            │  │
│  │                       │   │                            │  │
│  │  触发：编排层自动注入   │   │  触发：智能体主动判断调用   │  │
│  │  智能体：被动接收       │   │  智能体：主动决定是否用     │  │
│  │  配置方：系统设计者     │   │  配置方：用户（放文件即可） │  │
│  │  内容：结构化市场数据   │   │  内容：分析方法/领域知识    │  │
│  │  格式：标准化数值字段   │   │  格式：Markdown 说明文件   │  │
│  │                       │   │                            │  │
│  │  例：价格历史、技术指标 │   │  例：芯片行业分析框架       │  │
│  │      财务数据、资金流   │   │      财报季特殊分析方法     │  │
│  └───────────────────────┘   └───────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

## 三、类型A：数据工具（自动注入机制）

```
┌─────────────────────────────────────────────────────────────┐
│               数据工具工作流程                                │
│                                                             │
│  编排层启动某个智能体                                         │
│          ↓                                                  │
│  读取该智能体的数据工具配置（见 02文档）                       │
│          ↓                                                  │
│  按配置调用对应数据工具（传入：股票代码 + 时间范围）           │
│          ↓                                                  │
│  数据工具从数据层拉取已清洗、已计算好的数据                    │
│          ↓                                                  │
│  将数据 + 元信息（来源/时间/质量等级）注入智能体上下文         │
│          ↓                                                  │
│  智能体启动时，上下文已包含所需数据，直接开始分析              │
│  （智能体不知道数据从哪来，只看到内容本身）                   │
└─────────────────────────────────────────────────────────────┘
```

### 数据工具分类目录

```
数据工具按数据类型分类（与智能体无关，一类数据可被多个智能体使用）：

  price_tool          行情数据（开高低收/成交量/前复权）
  indicator_tool      技术指标（MACD/RSI/KDJ/布林带/均线系统）
  fundamental_tool    基本面数据（PE/PB/PS/财务三表摘要）
  capital_flow_tool   资金流向（主力/散户/北向净流入）
  margin_tool         融资融券数据
  dragon_tiger_tool   龙虎榜数据
  sentiment_tool      情绪数据（涨跌停统计/换手率）
  sector_tool         板块数据（行业指数/概念分类/动量因子）
  news_tool           新闻公告（新闻摘要/公告标题/停牌信息）
  snapshot_tool       价格快照（最新价/关键技术位/近期高低点）
  risk_metric_tool    风控指标（VaR结果/特有风险评分/波动率）
```

## 四、类型B：Skills 技能（主动调用机制）

```
┌─────────────────────────────────────────────────────────────┐
│               Skills 工作流程                                │
│                                                             │
│  系统启动 → 编排层扫描 skills/ 目录                           │
│          ↓                                                  │
│  将可用 Skill 列表告知智能体（文件名 + 适用场景描述）          │
│          ↓                                                  │
│  智能体分析过程中自行判断：当前任务是否需要某个 Skill？        │
│          ↓                                                  │
│  需要 → 调用该 Skill 文件内容，纳入分析流程                  │
│  不需要 → 不调用，继续正常分析                               │
│          ↓                                                  │
│  智能体在报告中注明：引用了哪个 Skill（透明性要求）            │
└─────────────────────────────────────────────────────────────┘
```

### Skills 目录结构

```
skills/                              ← 项目根目录下，非 docs/ 下
├── README.md                        ← 说明如何添加自定义 Skill
│
├── [行业专项]
│   ├── 芯片板块分析框架.md
│   ├── 新能源行业分析框架.md
│   └── 医药生物行业分析框架.md
│
├── [时间周期]
│   ├── 财报季分析重点.md
│   └── 年末资金博弈规律.md
│
└── [用户自定义]
    └── ...（用户随时添加，系统自动扫描）
```

### Skill 文件格式规范

```
每个 Skill 文件包含三个部分：

  ┌──────────────────────────────────┐
  │  头部元信息                       │
  │  - 技能名称                       │
  │  - 适用场景（智能体靠这个判断要不要用）│
  │  - 调用条件（什么情况下应该启用）    │
  └──────────────────────────────────┘
  ┌──────────────────────────────────┐
  │  正文：分析框架/方法/知识          │
  │  格式：标准 Markdown              │
  │  内容：具体如何分析、关注哪些指标   │
  └──────────────────────────────────┘
```

### Skills 使用规则

```
归属规则：
  Skills 不属于某个特定智能体
  所有智能体均可访问全部 Skills
  由各智能体自行判断是否相关并决定是否调用

添加规则：
  将 .md 文件放入 skills/ 目录 → 自动生效
  文件名要清晰（智能体靠文件名判断相关性）
  无需修改任何系统代码或配置文件
```

## 五、类型C：风控计算模块

```
┌─────────────────────────────────────────────────────────────┐
│               风控计算模块（非AI，纯代码逻辑）                 │
│                                                             │
│  位置：工具层内，但既不是数据工具也不是 Skill                  │
│  性质：确定性数学计算，没有AI判断                              │
│  触发：由编排层在 Stage 3 开始前自动调用                      │
│                                                             │
│  包含两个子模块：                                             │
│    模块1：VaR 计算（历史模拟法）                              │
│    模块2：A股特有风险评分（涨跌停/T+1/融资/停牌）             │
│                                                             │
│  计算结果通过 risk_metric_tool 注入 Stage 3 各风控智能体      │
│  详细规格见：docs/tools/03_风控计算模块.md                    │
└─────────────────────────────────────────────────────────────┘
```

## 六、三类工具对比速查

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│  对比维度     │  数据工具     │  Skills技能   │  风控计算模块 │
├──────────────┼──────────────┼──────────────┼──────────────┤
│  触发方式     │  编排层自动   │  智能体主动   │  编排层自动   │
│  内容类型     │  结构化数据   │  分析知识/方法 │  计算结果    │
│  智能体感知   │  无感知       │  有感知       │  无感知      │
│  可扩展方     │  系统设计者   │  用户         │  系统设计者  │
│  文件位置     │  代码内配置   │  skills/ 目录 │  代码内实现  │
│  详细配置     │  02文档       │  skills/README│  03文档      │
└──────────────┴──────────────┴──────────────┴──────────────┘
```
