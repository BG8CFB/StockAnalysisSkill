# 数据工具配置

> 版本：v1.0
> 所属层：工具层（三层架构第二层）
> 职责范围：定义每个智能体所需的数据工具及注入内容，是编排层数据注入的配置依据

---

## 一、工具动态激活机制（核心概念）

本文档定义的配置是**意向配置**，实际注入内容由以下流程动态决定：

```
┌───────────────────────────────────────────────────────────────────┐
│                    工具动态激活全流程                               │
│                                                                   │
│  Step 1: 读取智能体工具配置                                         │
│    每个智能体有独立的配置，声明"我需要哪些数据工具"                    │
│    示例：技术分析师需要 [price_tool, indicator_tool, sentiment_tool]  │
│                          ↓                                        │
│  Step 2: 检查数据源实际可用性                                        │
│    当前数据源（Tushare/AkShare）实际能返回哪些字段？                  │
│    依据：API 返回结果 + 积分等级 + 付费套餐                           │
│                          ↓                                        │
│  Step 3: 工具激活判断（取交集）                                      │
│                                                                   │
│    [智能体需要的工具]  ∩  [数据源可提供的工具]  =  [激活的工具]        │
│                                                                   │
│    差集部分（智能体需要但数据源无法提供）→ 该工具标记为"未激活"         │
│                          ↓                                        │
│  Step 4: 分两路注入智能体上下文                                      │
│                                                                   │
│    已激活工具  →  正常注入数据（含来源/时间/质量等元信息）              │
│    未激活工具  →  注入占位符说明（"XX数据不可用：原因"）               │
│                          ↓                                        │
│  Step 5: 智能体启动                                                 │
│    智能体看到上下文，对有数据的维度正常分析                             │
│    对有占位符的维度，按全局规则降级处理（标注N/A，权重重分配）           │
└───────────────────────────────────────────────────────────────────┘
```

### 灵活激活示例

```
情景：用户使用 Tushare 免费版（积分不足，无资金流接口）

  技术分析师配置：[price_tool ✅, indicator_tool ✅]
  → 正常运行，数据完整

  市场微观结构分析师配置：[capital_flow_tool ❌, dragon_tiger_tool ❌, margin_tool ❌]
  → capital_flow_tool 未激活（接口不可用）
  → 该智能体收到："主力资金数据不可用（数据源积分不足），本维度评分参考价值有限"
  → 智能体降级处理，评分中资金流维度标注 N/A

  结论：系统仍然可以运行，只是该维度的分析质量下降，透明告知用户
```

## 二、配置原则

- 每个智能体只注入其分析职责所需的数据，不注入冗余数据
- 数据范围（历史窗口）与智能体分析周期匹配
- 注入数据必须包含元信息（来源、时间、质量等级），供智能体在报告中引用
- 工具未激活时，注入标准占位符而非静默缺失，智能体必须感知并处理

## 二、Stage 1 数据工具配置

Stage 1 的六个分析师并行运行，各自注入不同侧重的数据。

### 2.1 技术分析师

**职责**：解读技术指标，判断趋势和关键点位

| 注入数据 | 内容说明 | 历史窗口 |
|---------|---------|---------|
| 行情数据（前复权） | 开高低收、成交量、成交额 | 250个交易日（覆盖年线计算） |
| 技术指标全量 | MACD / RSI / KDJ / 布林带 / 均线系统 / 成交量指标，含信号字段 | 最新计算结果 |
| 异常事件列表 | 近20日涨跌停、量价异常事件 | 近20日 |

### 2.2 基本面分析师

**职责**：解读财务数据和估值水平

| 注入数据 | 内容说明 | 历史窗口 |
|---------|---------|---------|
| 每日基本面指标 | PE/PB/PS/股息率/换手率/流通市值 | 最新值 + 近60日趋势 |
| 财务三表摘要 | 最近2期季报的营收/净利润/现金流关键指标 | 最近2个报告期 |
| 价值因子评分 | pe_ttm / pb_mrq / value_score | 最新值 |

### 2.3 市场微观结构分析师

**职责**：解读资金流向和筹码结构

| 注入数据 | 内容说明 | 历史窗口 |
|---------|---------|---------|
| 资金流向指标 | 当日/3日/5日/10日主力净流入、散户净流入、北向资金 | 近10日 |
| 资金流向评分 | capital_score / capital_signal | 最新值 |
| 龙虎榜数据 | 最近3次上榜记录（机构/游资方向） | 最近3次 |
| 融资融券数据 | 融资余额、占流通市值比例、5日变动幅度 | 近10日 |
| 涨跌停统计 | 近5日市场整体涨跌停家数（情绪参考） | 近5日 |

### 2.4 市场情绪分析师

**职责**：解读市场整体情绪和个股情绪

| 注入数据 | 内容说明 | 历史窗口 |
|---------|---------|---------|
| 市场情绪指标 | 上证/深证/创业板指数近期涨跌幅 | 近20日 |
| 市场涨跌停统计 | 全市场涨停家数、跌停家数（反映市场情绪温度） | 近10日 |
| 北向资金流向 | 北向净流入（大盘情绪参考） | 近10日 |
| 个股换手率 | 换手率及近20日均值对比 | 近20日 |
| RSI指标 | 个股RSI及超买超卖状态 | 最新值 |

### 2.5 板块轮动分析师

**职责**：判断个股所在板块的强弱和轮动位置

| 注入数据 | 内容说明 | 历史窗口 |
|---------|---------|---------|
| 概念/行业板块归属 | 该股所属的全部概念和行业分类 | 最新 |
| 板块指数行情 | 所属行业指数的近期涨跌幅 | 近20日 |
| 动量因子 | mom_1m / mom_3m / mom_6m / mom_3m_skip1m | 最新值 |
| 动量评分 | momentum_score | 最新值 |

### 2.6 资讯事件分析师

**职责**：解读新闻事件和公告对股价的潜在影响

| 注入数据 | 内容说明 | 历史窗口 |
|---------|---------|---------|
| 新闻资讯 | 近7日相关新闻摘要（若数据源支持） | 近7日 |
| 重大公告 | 近30日公司公告标题列表 | 近30日 |
| 停牌信息 | 是否有近期停牌记录及原因 | 近30日 |

**注意**：若新闻数据源不可用（需额外付费或接口不可达），该智能体收到的新闻字段为空，须在报告中注明"资讯数据不可用，本维度评分参考价值有限"。

## 三、Stage 2 数据工具配置

Stage 2 的智能体串行运行，主要输入来自 Stage 1 的分析报告（由编排层传递），辅以少量追加数据。

### 3.1 看涨分析师 / 看跌分析师

| 注入数据 | 内容说明 |
|---------|---------|
| Stage 1 六维分析报告 | 六个分析师的完整报告文本（由编排层传入） |
| 各维度评分汇总 | 6个维度的 0-100 评分及信号方向 |

### 3.2 研究主管

| 注入数据 | 内容说明 |
|---------|---------|
| Stage 1 六维分析报告 | 同上 |
| 各维度评分汇总 | 同上 |
| 多空辩论记录 | 看涨/看跌分析师的完整论点及反驳内容 |

### 3.3 交易计划师

| 注入数据 | 内容说明 |
|---------|---------|
| 研究主管裁决报告 | 投资方向、置信度、仓位建议、核心依据 |
| 当前行情快照 | 最新收盘价、关键支撑阻力位（来自技术分析师报告） |
| 市场规则 | 注入对应市场的交易规则（T+1/T+0/涨跌停等），由编排层按股票代码自动选择 |

## 四、Stage 3 数据工具配置

Stage 3 的风控智能体在纯代码计算完成后启动。

### 4.1 三位风控师（激进/保守/量化）

| 注入数据 | 内容说明 |
|---------|---------|
| 交易计划书 | Stage 2 交易计划师产出的完整交易参数 |
| VaR计算结果 | 工具层计算的 VaR 指标（见 03_风控计算模块.md） |
| A股特有风险评分 | 工具层计算的特有风险评分（仅A股） |
| 波动率因子 | volatility_20d / max_drawdown_20d / volatility_score |
| 融资融券风险数据 | margin_balance / margin_ratio / margin_change_5d |
| 市场规则 | 注入对应市场的交易规则 |

### 4.2 首席风控官

| 注入数据 | 内容说明 |
|---------|---------|
| 三位风控师的完整报告 | 激进/保守/量化风控师的分析报告（由编排层传入） |
| 交易计划书 | Stage 2 产出的原始交易参数 |
| 所有风控计算指标 | VaR / 特有风险评分 / 波动率 / 融资数据 |

## 五、Stage 4 数据工具配置

Stage 4 投资顾问在首席风控官裁决完成后启动，接收所有前序阶段的汇总数据，不再需要原始市场数据。

### 5.1 投资顾问

| 注入数据 | 内容说明 | 来源 |
|---------|---------|------|
| CRO最终裁决 | 批准/否决/有条件批准 + 最终交易参数（仓位/止损/持有期） | Stage 3 输出 |
| 交易计划书（完整版） | 入场区间、止损价、分批止盈目标、情景预案 | Stage 2 输出 |
| 六维评分汇总 | 六个分析师的 0-100 评分 + 信号方向（6行简表） | Stage 1 输出 |
| 价格快照 | 最新收盘价、今日涨跌幅、近5日最高/最低价 | 数据层 |
| 关键技术位 | 最近支撑位、最近阻力位、近20日日均振幅（用于价格区间计算） | 数据层（计算引擎） |
| 市场规则 | 对应市场的交易规则（T+1/T+0等） | 编排层按股票代码自动注入 |

**注意**：投资顾问无需原始K线数据，只需要计算好的关键位数值和最新价格快照。价格走势参考区间由投资顾问基于上述注入数据自行推算，不是数据层计算的。
